## Core Architecture: The GLM Family Innovation

GLM-4 is built upon the General Language Model (GLM) framework, which introduces a novel autoregressive blank infilling objective. Unlike GPT's unidirectional attention, GLM-4 employs a bidirectional attention mechanism for corrupted spans, combined with autoregressive generation for the rest [1]. This hybrid approach allows the model to capture both local and global context effectively.

### Key Architectural Details

- **Attention Mechanism:** GLM-4 uses a two-stream attention pattern. For non-corrupted tokens, it applies bidirectional attention; for corrupted spans, it uses autoregressive attention. This enables the model to handle both understanding and generation tasks without architectural changes.
- **Model Scale:** The largest GLM-4 variant has 130 billion parameters, with smaller versions (6B, 10B) also available [2]. The 130B model uses 96 transformer layers with a hidden size of 12,288.
- **Training Data:** Pre-trained on a massive corpus of 1.5 trillion tokens, including both Chinese and English text, with a significant focus on technical and scientific content [3].
- **Long Context Support:** GLM-4 supports a context window of up to 128K tokens, leveraging FlashAttention and ALiBi (Attention with Linear Biases) for efficient long-range dependencies [4].
